# 向社区下载中心提交你的项目

欢迎将你的 Termux 相关脚本、工具或项目提交到社区官网下载中心展示。

## 提交方式（二选一）

### 方式一：Fork + Pull Request（推荐）

适合熟悉 Git 的开发者，提交后自动进入审核队列。

1. **Fork 仓库**

   访问社区官网仓库，点击右上角 **Fork** 按钮，将仓库复制到你的账号下。

2. **克隆你的 Fork**

   ```bash
   git clone https://github.com/你的用户名/仓库名.git
   cd 仓库名
   ```

3. **创建新分支**

   ```bash
   git checkout -b add-project-你的项目名
   ```

4. **修改 `downloads-data.js`**

   这是唯一需要改动的文件。根据你的项目类型，在对应数组中添加数据：

   | 项目类型 | 添加位置 | 说明 |
   |----------|----------|------|
   | 脚本工具 | `scripts` 数组 | Shell/Python 等一键脚本 |
   | 项目 | `templates` 数组 | 开箱即用的完整项目 |
   | 更多资源 | `resources` 数组 | 工具推荐、官方资源等 |

   数据格式示例：

   ```javascript
   {
     title: "你的项目名称",
     desc: "一句话描述项目功能。",
     icon: "📦",
     url: "https://github.com/你的用户名/项目仓库",
   }
   ```

   字段说明：

   | 字段 | 必填 | 说明 |
   |------|------|------|
   | `title` | 是 | 项目名称，简洁明了 |
   | `desc` | 是 | 项目描述，50 字以内 |
   | `icon` | 是 | Emoji 图标，如 `📦` `🐍` `🔧` |
   | `url` | 是 | 项目链接 |

5. **提交并推送**

   ```bash
   git add downloads-data.js
   git commit -m "添加项目：你的项目名称（类型：脚本工具/项目/更多资源）"
   git push origin add-project-你的项目名
   ```

   **Commit 信息必须包含以下内容**：
   - 项目名称
   - 项目类型（脚本工具 / 项目 / 更多资源）
   - 简短说明

   示例：
   ```
   添加项目：Termux 一键备份脚本（类型：脚本工具）
   支持备份 Termux 环境到外部存储，包含软件包列表和配置文件。
   ```

6. **发起 Pull Request**

   回到你的 Fork 仓库页面，点击 **Compare & pull request**，填写 PR 描述：

   ```
   ## 提交项目

   - **项目名称**：你的项目名称
   - **项目类型**：脚本工具 / 项目 / 更多资源
   - **项目链接**：https://github.com/xxx/xxx
   - **改动文件**：仅 `downloads-data.js`
   ```

   提交后等待管理员审核合并。

---

### 方式二：向频道管理员发送消息

适合不熟悉 Git 的用户，手动提交等待管理员添加。

1. **加入 QQ 频道**

   点击官网底部或关于页面的「加入频道」链接，进入 **Termux 交流社区 QQ 频道**。

2. **联系管理员**

   在频道内找到管理员（带有管理员标识的用户），发送以下格式的消息：

   ```
   【提交下载中心项目】
   项目名称：你的项目名称
   项目类型：脚本工具 / 项目 / 更多资源
   项目描述：一句话描述项目功能
   项目链接
   图标 Emoji：📦
   ```

3. **等待审核**

   管理员收到消息后会进行审核，审核通过后会在 1-3 个工作日内添加到 `downloads-data.js` 中。审核结果会通过频道私信或公告通知。

---

## 审核标准

| 要求 | 说明 |
|------|------|
| 相关性 | 必须与 Termux 或 Android 终端开发相关 |
| 可用性 | 项目链接可正常访问，有基本的使用说明 |
| 安全性 | 无恶意代码、病毒或侵犯隐私的行为 |
| 原创性 | 优先接受原创项目， Fork 项目需注明来源 |

## 注意事项

- 仅改动 `downloads-data.js`，**不要修改 HTML 或 CSS 文件**
- 项目链接建议使用 GitHub，长期稳定可访问
- 一个 PR 建议只提交一个项目，多个项目请分批提交
- 如需更新已提交项目的信息，重新发起 PR 或联系管理员

---
